POWER-SMC MEDICAL REASONING — RESULTS & STATUS
================================================
Last updated: 2026-04-15

PROJECT GOAL
------------
Adapt Power-SMC (training-free inference-time reasoning) for medical VQA.
Test whether Power-SMC provides meaningful accuracy gains over baselines
(greedy, temperature sampling) on VQA-RAD (451 test questions, radiology).

THESIS
------
Power-SMC amplifies "correct knowledge" already in the model via
importance-weighted resampling during generation. The gain should be
larger for models with stronger domain knowledge (medical-specialized
VLMs) and for tasks requiring multi-step reasoning (CoT).

========================================================================
KEY FINDINGS
========================================================================

1. POWER-SMC GAIN IS SMALL ACROSS THE BOARD (~1pp)
   - Qwen CoT: +1pp
   - LLaVA-Med direct: +0.1pp
   - The +11pp LLaVA-Med CoT result was an ARTIFACT: CoT degrades
     LLaVA-Med's baseline more than SMC can recover, so the "gain" was
     just SMC rescuing a bad prompt, not genuine improvement.

2. COT HURTS MODELS NOT TRAINED FOR IT
   - LLaVA-Med greedy: 21.9% (CoT) vs 38.8% (direct) — CoT costs 17pp
   - LLaVA-Med was instruction-tuned on PubMed for direct Q->A,
     not multi-step reasoning traces.

3. QWEN > LLAVA-MED DESPITE BEING "GENERAL-PURPOSE"
   - Qwen best: 46.5% vs LLaVA-Med best: 38.9%
   - LLaVA-Med v1.5 is from 2023 (Mistral-7B base), Qwen2.5-VL is
     2025 with stronger vision encoder and reasoning training.

4. POWER-SMC NEEDS LONG REASONING TO STEER
   - With direct prompts, model emits 1-5 tokens. No reasoning trace
     for SMC to resample over. SMC's value is amplifying good reasoning
     PATHS, which requires sequences long enough to diverge.
   - Original Power-SMC paper showed gains on MATH500 with 200-500
     token solutions.

5. OPEN-ENDED QUESTIONS ARE HARD FOR ALL METHODS
   - Best open accuracy: ~26% (Qwen). String-match grading is strict
     but not the main bottleneck.


========================================================================
BUGS FIXED (in order)
========================================================================

1. LLaVA-Med baseline kwarg: generate() needs `inputs=` not `input_ids=`
   - LLaVA's generate() reads tokens from positional `inputs` arg
   - Commit: 33c1d2f

2. LLaVA-Med baseline output trimming: set prompt_len=0
   - LLaVA's generate() returns ONLY new tokens (not prompt+new)
   - Eval script's generated_ids[0, prompt_len:] was trimming everything
   - Commit: 3a2ae66

3. LLaVA-Med conv template: USER:/ASSISTANT: -> mistral_instruct
   - Model trained with [INST]...[/INST] format
   - Wrong template costs 10-20pp
   - Commit: bfb608e

4. LLaVA-Med image tokenization: plain tokenizer() -> tokenizer_image_token()
   - <image> must be replaced with IMAGE_TOKEN_INDEX (-200) before
     embedding, so prepare_inputs_labels_for_multimodal can splice
     CLIP features at that position
   - Plain tokenizer encoded <image> as literal characters
   - Same commit as #3: bfb608e

5. LLaVA-Med image preprocessing: manual preprocess -> process_images()
   - Honors model.config.image_aspect_ratio ("pad" for v1.5)
   - Same commit: bfb608e

6. Power-SMC attention mask mismatch for multimodal models
   - LLaVA expands 1 <image> token into ~576 CLIP tokens during forward
   - KV cache length (649) != input_ids length (74) after prefill
   - Decode loop sized attn_buf from input_ids length -> mask mismatch
   - Fix: get_kv_seq_length() reads actual cache length after prefill
   - Applied to both generate() and generate_batch()
   - Commit: 25518c6

NEXT STEPS
========================================================================

1. MED-GEMMA RESULTS (in progress)
   - Job running. Med-Gemma supports reasoning, so CoT might actually
     help here unlike LLaVA-Med. If SMC shows a gain with Med-Gemma CoT,
     that supports the thesis that SMC needs reasoning traces to steer.

2. HARDER DATASET
   - VQA-RAD questions are often simple (yes/no about visible features).
     Answers are 1-5 tokens. Power-SMC needs longer reasoning chains.
   - Candidates: PathVQA (more complex pathology), PMC-VQA (PubMed
     figures), SLAKE (bilingual medical VQA)
   - A dataset requiring multi-step diagnostic reasoning would be ideal.

3. PROMPT ENGINEERING
   - Test explicit multi-step reasoning prompts that force longer outputs
   - "First describe what you see, then identify abnormalities, then
     answer the question" style prompts

4. HYPERPARAMETER SWEEP (already done for Qwen, 100 questions)
   Results in results/sweep/. Best config: alpha=4.0, N=16, kappa=0.5 (52.5%)

   Config                      | Overall | Closed | Open
   alpha=2.0, N=16, kappa=0.5  | 49.0%   | 61.0%  | 31.7%
   alpha=4.0, N=16, kappa=0.3  | 48.5%   | 59.3%  | 32.9%
   alpha=4.0, N=16, kappa=0.5  | 52.5%   | 64.4%  | 35.4%  ** best **
   alpha=4.0, N=16, kappa=0.7  | 47.8%   | 59.3%  | 31.1%
   alpha=4.0, N=32, kappa=0.5  | 50.5%   | 60.6%  | 36.0%
   alpha=4.0, N=64, kappa=0.5  | 49.0%   | 61.9%  | 30.5%
   alpha=6.0, N=16, kappa=0.5  | 50.5%   | 61.0%  | 35.4%
   alpha=8.0, N=16, kappa=0.5  | 45.5%   | 57.6%  | 28.0%

   Takeaways: alpha=4 is the sweet spot. More particles (32, 64) don't help.
   kappa=0.5 (resample at 50% ESS) is optimal. High alpha (8) hurts.
